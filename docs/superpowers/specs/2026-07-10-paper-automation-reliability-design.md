# 模拟盘自动化可靠性改造设计

## 背景

2026-07-08 的云端日报显示扫描在 `238 只 / 20 批` 时结束，状态为未完成，模拟盘没有执行。现网核查表明定时器和持仓数据库正常，故障集中在候选扫描编排。

当前编排每批重新请求一次全市场快照，关闭扫描缓存，并且只允许 `20 x 40 = 800` 次扫描额度。任意一次数据源降级都会消耗批次但不推进游标。扫描未完成时，持仓止损和利润保护也不执行。

## 目标

1. 无论候选扫描是否完成，已有持仓都必须每日执行行情刷新、止损、利润保护和退出检查。
2. 新开仓只能在当日 800 只候选扫描完成后执行。
3. 全市场快照每个交易日只抓取一次，后续所有批次使用同一份排序后候选池。
4. 失败请求不消耗成功批次，支持有上限的重试和断点续扫。
5. 日报明确显示成功批次、失败尝试、最后游标、数据源错误以及风控是否执行。

## 方案比较

### 方案 A：仅提高最大批次

将 20 批提高到 30 或 40 批。改动小，但仍会每批重抓全市场，也没有解决持仓风控被扫描阻断的问题。不采用。

### 方案 B：固定快照与风控解耦

当日首次成功抓取全市场快照后，保存入盘，固定市值前 30% 和预筛后 800 只名单。后续只用腾讯 K 线逐批补日线。扫描失败时仍运行持仓风控，但禁止新开仓。这是本次采用的方案。

### 方案 C：完整本地行情中台

以沪深北官方股票池为主表，每日全量更新腾讯 K 线，另建市值、PE 和股本变化快照。长期最稳定，但本次范围和数据治理成本过大，作为后续项目。

## 架构与数据流

### 1. 当日扫描快照

新增独立的扫描快照文件，保存：

- 交易日和抓取时间；
- 快照提供方和告警；
- 全市场数量、市值前 30% 数量；
- 预筛后 800 只 `SpotStock` 必要字段。

快照只在日期、策略版本和股票池口径一致时可复用。新交易日或策略版本变更必须重建。

### 2. 分批精筛

后台每批从快照中按游标取 40 只，腾讯 K 线作为日线主源。东方财富和新浪仅用于建立全市场快照及补充市值、PE 等字段。

只有游标前进才计为成功批次。数据源降级、请求异常或游标不变计为失败尝试。默认允许 10 次失败尝试，失败后等待 3 秒再继续，防止短时限流。

### 3. 持仓风控与候选开仓

模拟盘执行增加两种模式：

- `full`：执行持仓退出和新开仓，仅在扫描完成时使用。
- `risk_only`：只执行持仓止损、利润保护和卖出，候选列表为空，扫描未完成时使用。

当天首先生成 `risk_only` 复盘后，若后续补扫完成，新的扫描 `updatedAt` 晚于复盘时间，现有去重逻辑允许再执行一次 `full` 判断。已卖出持仓已从账户移除，不会重复卖出；同日卖出代码仍受现有“避免回补”规则保护。

### 4. 日报与日志

日报状态分为：

- 成功：扫描完成且 `full` 执行完成。
- 部分完成：扫描未完成，但 `risk_only` 持仓风控已执行。
- 失败：持仓风控本身也未执行成功。

日报和 JSON 日志保存成功批次、失败尝试、总尝试、游标、已分析数、扫描告警、模拟盘模式、仓位、交易数和错误原因。任务返回前尽量发送日报，不再因单个扫描批次抛错而整个退出。

## 异常处理

- 当日全市场快照未能建立：不开新仓，执行 `risk_only`，保留上次快照作为展示但不用于当日开仓。
- 腾讯日线单批失败：不推进游标，记录失败并重试。
- 腾讯日线部分股票缺失：批次仍可推进，记录缺失代码，不伪造样本。
- 持仓实时价失败：使用最后有效行情快照，在日报中标记估值模式。
- 持仓风控执行失败：状态标记为失败，仍发送带错误原因的企业微信日报。

## 数据源决策

腾讯 `newfqkline` 接口继续作为 800 只日线精筛的主源。它不直接替换全市场快照，原因是缺少稳定的市值和 PE 字段，并且 2026-07-10 全量 5530 只试跑在第八批后明显退化，总成功率为 69.46%。

本次通过固定 800 只范围、降低请求量、限制并发、失败重试和持久化快照使用该接口。沪深北官方股票池与完整本地行情中台作为后续独立改造。

## 测试策略

1. 扫描批次连续失败后成功，确认失败不消耗成功批次且最终执行 `full`。
2. 失败达上限，确认执行 `risk_only`，有风控卖出但无买入。
3. 全市场快照在多批扫描中只请求一次，重启后能从同日磁盘快照续扫。
4. 快照日期或策略版本不匹配时不得用于当日开仓。
5. 精筛中部分日线缺失可记录并继续，整批异常必须重试。
6. 风控失败时仍生成可诊断日报。
7. 完整测试集、TypeScript 构建和现网 API 回归全部通过。

## 本次不做

- 不改变买入规则、止损参数、利润保护参数和仓位公式。
- 不将腾讯公开接口标记为官方数据源。
- 不在本次建立 5530 只全量历史行情中台。
- 不增加新的付费数据供应商。
- 不修改每日 15:00 的主任务时间。
